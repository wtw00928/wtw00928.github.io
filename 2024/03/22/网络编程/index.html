网络编程

1、网络编程以及通信协议是什么？

网络编程是指编写程序以实现计算机网络通信的过程。在网络编程中，程序可以通过网络连接与其他计算机或设备进行通信，传输数据、接收数据，并进行各种网络相关的操作。

通信协议其实就是计算机间互相通信所要遵守的规则。

2、URL、IP、HTTP、Socket以及端口是什么？

URL其实就是我们经常说的网址：https://wtw00928.github.

IP地址分为IPv4和IPv6两种。IPv4采用32位地址，类似101.202.99.12，而IPv6采用128位地址，类似2001:0DA8:100A:0000:0000:1020:F2F3:1428，其中101表示的是一个8位二进制。

HTTP(Hypertext Transfer Protocol，超文本传输协议)就是一个通信协议。HTTPS是HTTP的安全版本，通过在HTTP上加入SSL/TLS协议层来加密通信内容，从而提供更高的安全性。

端口就是计算机与服务器进行双向通信的逻辑通道，可以想象因为一个计算机内需要运行多个程序，所以端口的数量也很多1~65535。

Socket是用于描述通信端点的抽象，它包含了IP地址和端口号。在建立网络连接时，通常会指定本地端口和远程端口。本地端口是指当前计算机上的应用程序使用的端口，远程端口是指通信对方的计算机上的应用程序使用的端口。当两个应用程序需要进行通信时，它们分别创建自己的Socket，并通过网络连接到对方的Socket，从而建立通信链路。

3、细谈Socket

Socket类的构造函数通常有两种形式，用于创建客户端和服务器端的套接字。以下是这两种形式的概述：

  1. 客户端Socket的构造函数：

    这个构造函数用于创建一个客户端套接字，并指定要连接的服务器的主机名（或IP地址）和端口号。它会尝试与指定的主机和端口建立连接。

    “localhost” 是一个特殊的主机名，通常用来引用本地计算机上的网络接口。它对应着本地回环地址（loopback address）127.0.0.1

  2. 服务器端Socket的构造函数：

    这个构造函数用于创建一个服务器端套接字，并绑定到指定的端口上，以便监听客户端的连接请求。当客户端尝试连接到指定端口时，服务器端套接字将接受连接请求并创建一个新的Socket对象，用于与客户端进行通信。

    DatagramSocket 是 Java 中用于发送和接收 UDP 数据包的类。它允许应用程序通过 UDP 协议在网络上发送数据，并从网络中接收数据

###4、DatagramSocket是什么？

DatagramSocket 是 Java 中用于发送和接收 UDP 数据包的类。

DatagramSocket相当于一个码头，DatagramPacket相当于一个集装箱。集装箱里装有数据，码头将集装箱的数据发送到指定集装箱的目标地址和端口中。

5、网络编程中的服务器端与客户端

编写程序时一般先看服务端，再看客户端，因为服务端要一直监听是否由客户端连接。服务端只要端口号，不用IP，而客户端需要服务端的IP，以及端口号，

  1. 服务器端：
    • 服务器端是网络服务的提供者，它通常在服务器上运行，等待客户端的连接请求。
    • 服务器端负责接收客户端的连接请求，并为每个连接请求创建一个新的会话（或称为会话、连接、套接字）。
    • 一旦建立了连接，服务器端负责处理客户端发送过来的请求，并提供相应的服务或资源。
  2. 客户端：
    • 客户端是网络服务的使用者，它通常在用户的计算机上运行，向服务器端发送请求并接收响应。
    • 客户端负责与服务器端建立连接，并发送请求数据。
    • 客户端接收服务器端返回的响应，并根据需要对响应进行处理和显示给用户。
    • 客户端可以与多个服务器端进行通信，但通常一次只与一个服务器端建立连接
